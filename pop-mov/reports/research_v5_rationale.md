# 🔬 v5改善根拠記録（エビデンスベース）

> **目的**: 各改善の根拠を記録し、「なぜ変えるのか」を誰でも検証可能にする

## 📌 改善記録のルール

1. 各変更は **数値根拠** を伴うこと
2. 人間の **知覚閾値** を明記すること
3. 改善前/後の **比較値** を記録すること
4. 検証可能な **測定方法** を記載すること
5. 失敗した変更も **教訓として記録** すること

## P0修正: 1行変更で即効果（5件）

---

### P0-1: ビット同期ズーム量

| 項目 | 値 |
|---|---|
| **現在** | `ZOOM_AMOUNT = 0.015` (1.5%) |
| **変更後** | `ZOOM_AMOUNT = 0.05` (5.0%) |
| **根拠** | 人間の視覚が知覚できる最小のサイズ変化は **約2-3%**（Brenner & Smeets, 1996）。1.5%は閾値未満で事実上「変更なし」と同等 |
| **参考** | 映画/TVのプロダクションでは 3-8% のdolly zoomが一般的。5%は「無意識に感じるが明確に変化している」レベル |
| **BPM同期** | 128 BPM = 2.13Hz → 0.469秒周期。このリズムで5%ズームイン/アウトは subconscious entrainment（無意識の同調）を引き起こす十分な強度 |
| **検証方法** | ズームあり/なしの2バージョンを10人に盲検で提示。「何か動きがある」と感じた人の割合を測定。5%で60%以上が知覚すれば成功 |
| **予測効果** | +3点（視覚リズム知覚の向上） |
| **リスク** | 10%以上だと「酔い」の原因。5%は安全域 |

---

### P0-2: ゴールデンフラッシュ透明度

| 項目 | 値 |
|---|---|
| **現在** | `opacity=20` (約8%、RGBA 20/255) |
| **変更後** | `opacity=64` (約25%、RGBA 64/255) |
| **根拠** | 白色背景(RGB 255,255,255)上にゴールド(RGB 255,215,0)をオーバーレイする場合、ウェーバーの法則（ΔI/I）によれば明るさの差が約10%以上で知覚可能。8%では ΔB = (255-215)/255 × 0.08 = 1.3% → 知覚不能。25%では ΔB = 3.9% → 境界値、30%では ΔB = 4.7% → 知覚可能 |
| **色彩心理学** | ゴールド（#FFD700）はドーパミン放出を促進する「報酬色」。研究では0.1-0.2秒の提示でポジティブ感情の増加が確認されている（Gorn et al., 2004） |
| **タイミング** | t=2.5秒は各シーンの「アハ！」瞬間（メイン情報の提示が完了した直後）。ここでゴールドを表示することで「発見の喜び」を視覚的に補強 |
| **検証方法** | フラッシュあり/なしを10人に提示。「何か光った」と感じた人の割合。25%で50%以上、30%で70%以上が知覚すれば成功 |
| **予測効果** | +2点（報酬感覚の向上） |
| **リスク** | 50%以上だと「画面がチカチカする」不快感。25-30%が安全域 |

### P0-3: Ding SEの長さ

| 項目 | 値 |
|---|---|
| **現在** | `duration=0.1` (100ms) |
| **変更後** | `duration=0.2` (200ms) |
| **根拠** | 人間の聴覚が「音」として知覚できる最小長は約 **50-80ms**（ただし「何の音か」を識別するには150-200ms必要）。100msは「カチッ」というクリック音にしか聞こえない。200msなら1200Hzのサイン波が約2.4周期含まれ、「キン」という金属音として識別可能 |
| **周波数** | 1200Hzは人間の聴覚が最も敏感な 1000-4000Hz 帯域内。等ラウドネス曲線（ISO 226:2003）によれば、この帯域は同じ音量で最も大きく聞こえる |
| **エンベロープ** | `exp(-t * 40)` の減衰: 0.1秒版では t=0.1 で exp(-4) = 0.018 → ほぼ無音。0.2秒版では t=0.2 で exp(-8) = 0.0003 → 依然として急減衰だが、最初の50msにエネルギーが集中するため「キン」というアタック感が明確になる |
| **検証方法** | 100ms版と200ms版をABXテスト（どちらか判別）。「金属音として聞こえた」割合を測定。200msで70%以上が「キン」と認識すれば成功 |
| **予測効果** | +2点（reveal momentの強調） |
| **リスク** | 0.3秒以上だと「長くて邪魔」。0.2秒が最適 |

### P0-4: Whoosh SEを全遷移で再生

| 項目 | 値 |
|---|---|
| **現在** | パターンインターラプト時のみ（1回） |
| **変更後** | 各シーン遷移時（3回: シーン1→2, 2→3, +パターンインターラプト） |
| **根拠** | 動画編集の業界標準: シーン切り替えには必ず遷移SEを付ける。1回だけだと「なぜここで鳴るのか」が不明確になり、かえって不自然。全遷移で一貫性を持つことで「プロダクション品質」が上がる |
| **タイミング** | 各シーンの終わり（t=4.85-5.0s, 9.85-10.0s）+ パターンインターラプト（t=9.85-10.0s） |
| **音量** | 0.12（BGM 0.15より少し下、音声 0.9より十分に下） |
| **検証方法** | Whoosh 1回版と3回版を比較。「プロに見えた」割合を測定。3回版で60%以上が「統一感がある」と評価すれば成功 |
| **予測効果** | +3点（一貫性のある音響設計） |
| **リスク** | 音量を大きくしすぎると主音声と競合。0.12以下を維持 |

### P0-5: フックフラッシュ時間

| 項目 | 値 |
|---|---|
| **現在** | `duration=0.15` (150ms) |
| **変更後** | `duration=0.3` (300ms) |
| **根拠** | 人間の視覚処理: 完全な赤画面のフラッシュは100ms以下だと「チラつき」に過ぎない。200-300msで「意図的なエフェクト」として認識される。Orienting response（定向反応）を引き起こすには 200ms以上が必要（Sokolov, 1963） |
| **色彩** | 純赤 (255, 0, 0) は最も強い警告色。0.3秒ならスクロールを止めるには十分、不快になるには短い |
| **検証方法** | 0.15s版と0.3s版を比較。「attentionを引いた」割合。0.3sで70%以上が「ハッとした」と評価すれば成功 |
| **予測効果** | +3点（フック強度の向上） |
| **リスク** | 0.5秒以上だと「うるさい/攻撃的」。0.3秒が最適 |

## P1追加: 新機能（3件）

---

### P1-1: テキストscale-inアニメーション

| 項目 | 値 |
|---|---|
| **現在** | テキストはフェードインのみ（alpha 0→255） |
| **追加** | scale 0.3→1.0 のelastic overshoot（1.1→1.0） |
| **根拠** | モーショングラフィックスの原則: 要素が「外から入ってくる」ことで注意を引く。フェードインは「出現」だが、スケールインは「誕生」の感覚を与える。elastic overshoot（行き過ぎ→収束）は物理的なリアリティを感じさせる |
| **参考** | AppleのHuman Interface Guidelines、Material Designのモーション仕様はともに scale-in + overshoot を推奨 |
| **実装** | MoviePyの `fl` で `resize` を時間関数化: `scale(t) = 0.3 + 0.8 * ease_out_elastic(t/duration)` |
| **検証方法** | フェードイン版とスケールイン版を比較。「印象的だった」割合 |
| **予測効果** | +3点 |

---

### P1-2: ミニフックを各シーンで文言変更

| 項目 | 値 |
|---|---|
| **現在** | 全シーン共通で「実は…」（t=2.5s） |
| **追加** | シーン固有の文言: カタツムリ「仲間外れ!?」、猫「25Hzの秘密」、ハチドリ「超人すぎる…」 |
| **根拠** | パターン認識の原則: 同じ文言を3回繰り返すと「テンプレート感」が出て手抜きに見える。各シーン固有の文言は「それぞれのネタに個別の注意を払っている」印象を与える |
| **検証方法** | 共通文言版と固有文言版を比較。「各ネタに個性があった」と感じた割合 |
| **予測効果** | +2点 |

---

### P1-3: イラストpulseを全シーンで実装

| 項目 | 値 |
|---|---|
| **現在** | シーン0のみ（snail）でpulse実装 |
| **追加** | 全3シーンでpulse（t=2.5s, scale 1.0→1.05→1.0） |
| **根拠** | 一貫性の原則: 1つのシーンだけエフェクトがあると「バグ」か「意図的か」の判断がつかない。全シーンで同じエフェクトを使うことで「デザインシステム」として成立する |
| **pulse量** | 5%スケールUP。これはP0-1のズームと同じ根拠（2-3%以上で知覚可能） |
| **検証方法** | pulseなし/1シーン/全シーン を比較。「生き生きとしていた」と感じた割合 |
| **予測効果** | +2点 |

## 📊 v4失敗の教訓記録

---

### 教訓1: 「数」≠「効果」

| 項目 | 内容 |
|---|---|
| **v4の主張** | 「11のマイクロアニメーション」 |
| **実際** | 機能したのは2-3個のみ |
| **原因** | 各アニメーションの強度が知覚閾値未満だった |
| **教訓** | 「3個がちゃんと見える」> 「11個が見えない」 |

---

### 教訓2: 知覚閾値を意識せよ

| エフェクト | v4設定値 | 知覚閾値 | 結果 |
|---|---|---|---|
| ズーム量 | 1.5% | 2-3% | ❌ 見えない |
| フラッシュ透明度 | 8% | 10%+ | ❌ 見えない |
| Ding SE長さ | 100ms | 150-200ms | ❌ 聞こえない |
| フックフラッシュ | 150ms | 200ms+ | ❌ 短い |

---

### 教訓3: 一貫性が品質を決める

| 項目 | v4 | 問題 |
|---|---|---|
| Whoosh SE | 1回のみ | なぜここでだけ？という疑問 |
| イラストpulse | 1シーンのみ | バグに見える |
| **教訓** | 使うなら全箇所で、使わないなら nowhere | 中途半端が一番悪い |
