# 🧠 認知科学に基づく動画分析フレームワーク

> 15秒Shorts動画の全要素を認知科学の知見で検証・改善する

---

## 📑 認知科学マップ

```
【注意 Attention】
├── 定向反射 (Orienting Response) — 0.3秒
├── 選択的注意 (Selective Attention) — 3秒フィルター
├── サリエンシー・マップ (Saliency Map) — 視覚的突出
└── 注意の瞬き (Attentional Blink) — 200-500msの死角

【知覚 Perception】
├── ウェーバーの法則 (Weber's Law) — 最小知覚差 (JND)
├── ゲシュタルトの法則 — 群化・閉合・連続性
├── 時間知覚 — 100ms/1s/3sの階層
└── 色彩知覚 — 補色・暖色/寒色

【記憶 Memory】
├── 系列位置効果 (Serial Position Effect) — 初頭効果/親近効果
├── フォン・レストルフ効果 — 孤立化されたものは記憶される
├── ピークエンドの法則 — ピークと終わりで評価
└── 作業記憶容量 — ミラーの法則 (7±2)、実際は4チャンク

【感情 Emotion】
├── アrousal-Valence モデル — 高 arousal が記憶に残る
├── 予期と報酬 — ドーパミンは「予測誤差」で放出
└── 感情の伝染 — 音声の感情が視聴者に転移

【学習 Learning】
├── 認知負荷理論 (Cognitive Load Theory) — 内在/外在/本質的負荷
├── 二段処理理論 — 自動処理 vs 制御処理
└── ス pacing 効果 — 情報は間隔を空ける
```

## 🔬 各法則の動画への適用

---

### 1. 定向反射 (Orienting Response)

| 項目 | 内容 |
|---|---|
| **提唱者** | Sokolov (1963) |
| **知見** | 突然の変化（光・音・動き）に脳が自動的に注意を向ける。反射は **200-500ms** で発生 |
| **閾値** | 視覚: 200ms以上の提示 / 聴覚: 80dB以上の急変化 |
| **v4での失敗** | フックフラッシュ 150ms → **閾値未満**。知覚されなかった |
| **v5修正** | 赤フラッシュ **300ms** + Whoosh SE 80dB換算 |
| **設計原則** | 「最初にハッと思わせる」ものを 0.3秒で提示 |

**動画適用マップ:**
```
t=0.00s: 赤フラッシュ (300ms) → Orienting Response 発動
t=0.30s: フックテキスト表示 → 注意がフックに向けられる
t=5.00s: Whoosh SE → シーン2への注意リセット
t=10.00s: Whoosh SE → シーン3への注意リセット
```

### 2. ウェーバーの法則 (Weber's Law)

| 項目 | 内容 |
|---|---|
| **提唱者** | Weber (1834), Fechner (1860) |
| **知見** | 刺激変化の最小知覚差 (JND) は背景強度に比例。ΔI/I = k |
| **視覚サイズJND** | k ≈ 0.02-0.03 (2-3%) |
| **明度JND** | k ≈ 0.08 (8%) |
| **色相JND** | k ≈ 0.05 (5%) |
| **聴覚周波数JND** | k ≈ 0.003 (0.3%) @ 1000Hz |
| **聴覚音量JND** | k ≈ 0.1 (10%) = 約1dB |

**v4失敗分析:**
| エフェクト | v4値 | JND閾値 | 結果 |
|---|---|---|---|
| ズーム量 | 1.5% | 2-3% | ❌ 知覚不能 |
| ゴールデンフラッシュ | 8% (RGBA 20/255) | 明度JND 8%だが白色背景上でゴールドはΔB不足 | ❌ 境界値 |
| Ding SE | 100ms | 150-200ms (音識別) | ❌ 識別不能 |

**v5設計値:**
| エフェクト | 計算 | v5値 | JND比 |
|---|---|---|---|
| ズーム量 | 3% (JND下限) × 1.67 | **5%** | 1.67x JND |
| フラッシュ透明度 | 8% (JND) × 3.1 | **25%** | 3.1x JND |
| ゴールドオーバーレイ | ΔB = (255-215)/255 × α ≥ 8% → α ≥ 51% → 余裕を持って | **25%** (暗い部分では知覚可能) | - |

---

### 3. 系列位置効果 (Serial Position Effect)

| 項目 | 内容 |
|---|---|
| **提唱者** | Ebbinghaus (1885), Murdock (1962) |
| **知見** | 系列の**最初**（初頭効果）と**最後**（親近効果）の情報が最も記憶に残る。中央は最も忘れられる |
| **再現パターン** | U字型の記憶保持カーブ |

**v4のシーン順設計（修正済み）:**
```
v2/v3: Snail → Hummingbird → Cat (事実の衝撃度順)
v4:    Snail → Cat → Hummingbird (盛り上がり構造)
       ↑初頭効果        ↑親近効果（最も衝撃）
```

**認知科学的評価:**
- ✅ Hummingbird（1秒80回）を最後に → **親近効果**で最も印象に残る
- ✅ Snail（仲間外れ）を最初に → **初頭効果**で第一印象を決定
- ⚠️ Cat（25Hz修復）が中央 → 系列位置効果で最も忘れられやすい
  → 対策: ミニフック「25Hzの秘密」で注意を再誘導

**改善提案:**
- 中央のネタは **独立した特徴**（Von Restorff効果）で記憶に残す
- 「実はこの真ん中のネタが一番すごいんだけど」と明示的に宣言

---

### 4. フォン・レストルフ効果 (Von Restorff Effect)

| 項目 | 内容 |
|---|---|
| **提唱者** | Von Restorff (1933) |
| **知見** | 同種の刺激の中で **孤立して際立ったもの** が最も記憶される |
| **条件** | 周囲との差異が明確であること。文脈内での「逸脱」が必要 |

**v4での適用状況:**
| 要素 | 均質性 | 逸脱 | 効果 |
|---|---|---|---|
| ツッコミバルーン色 | ❌ 全シーン赤（v3）→ ✅ 赤/黄/青（v4） | ✅ 各シーンで補色 | ✅ 機能している |
| ミニフック文言 | ❌ 全シーン「実は…」 | ❌ 差異なし | ❌ Von Restorff なし |
| イラストスタイル | ✅ 全シーン統一スタイル | - | ✅ 一貫性はあるが差異なし |

**v5改善:**
- ミニフックを各シーンで**文脈固有**に変更 → 各シーン内で孤立化
  - Snail: 「仲間はずれ…だって😢」
  - Cat: 「25Hzがスゴイ」
  - Hummingbird: 「疲れない理由」
- 各ミニフックは **周囲のテキストと色を変える**（白→アクセントカラー）

### 5. ピークエンドの法則 (Peak-End Rule)

| 項目 | 内容 |
|---|---|
| **提唱者** | Kahneman et al. (1993) |
| **知見** | 体験の記憶は **最も強烈な瞬間（ピーク）** と **終わりの印象** で評価される。全体の平均ではない |

**v4のピーク/エンド分析:**
```
ピーク候補:
├── t=0.0-0.3s: 赤フラッシュ → 高arousalだが短すぎた
├── t=2.5s: ゴールデンフラッシュ → 知覚されなかった
├── t=3.3s: ツッコミSE+バルーン → 中程度
├── t=10.0s: ハチドリ「1秒80回」→ 最大の事実衝撃 ★
└── t=14.5s: ループフック → 低arousal

エンド:
└── t=14.5-15.0s: 「まだある↓」→ 低arousal、継続意図はあるが感情インパクトなし
```

**v5改善:**
- **ピーク強化**: ハチドリシーンで追加エフェクト
  - テキスト「1秒80回」を72px → **84px** にUP
  - ゴールデンフラッシュを25% → **40%** にUP（ハチドリシーンのみ）
  - Ding SEをハチドリシーンでも使用（reveal moment）
- **エンド強化**: ループフックを改善
  - 「まだある↓」→ **「もっとある▶ 保存してね」**
  - 背景をピンク (233,30,99) → **オレンジ (255,152,0)** に変更（暖色で高arousal）

### 6. 認知負荷理論 (Cognitive Load Theory)

| 項目 | 内容 |
|---|---|
| **提唱者** | Sweller (1988) |
| **知見** | 作業記憶容量は有限。**本質的負荷**（学習内容）+ **外在的負荷**（提示方法）+ **内的負荷**（要素間関係）の合計が容量を超えると学習不能 |
| **作業記憶容量** | 4チャンク（Cowan, 2001）— ミラーの7±2は更新済み |

**v4の1画面認知負荷分析:**
```
シーン1 (t=0.45-2.5s) の同時表示要素:
1. 背景グラデーション
2. 番号バッジ「1」
3. カタツムリイラスト
4. イラストシャドウ
5. メインテロップ「巻き方で仲間外れ!?」
6. 説明テキスト（3行）
7. プログレスバー
8. ビット同期ズーム（無意識）
→ 合計8要素 → 4チャンクを大幅超過 ❌
```

**v5改善（認知負荷削減）:**
- プログレスバーを **番号バッジに統合**（1要素にまとめる）
- 説明テキストを **1行に短縮**（3行→1行）
- イラストシャドウを **50%削減**（視覚的ノイズ低減）
- 目標: **5要素以下** に削減

### 7. 予期と報酬 — ドーパミン予測誤差

| 項目 | 内容 |
|---|---|
| **提唱者** | Schultz et al. (1997) |
| **知見** | ドーパミンニューロンは **予測誤差**（期待 > 結果 = ネガティブ、期待 < 結果 = ポジティブ）で発火。予想外の良い結果で最大放出 |
| **応用** | 「意外な事実」→ 期待を裏切る情報量が多いほどドーパミン放出 → 記憶定着 |

**v4の事実驚訝度分析:**
| ネタ | 意外性 | 予測誤差 | 評価 |
|---|---|---|---|
| カタツムリ「仲間認識されない」 | 中 | 中 | △ |
| 猫「25Hzで細胞修復」 | 高 | 高 | ○ |
| ハチドリ「1秒80回で疲れない」 | **非常高** | **非常高** | ◎ |

**v5改善:**
- 各ネタの前に **期待形成** を入れる
  - Snail: 「この生き物、意外とシビア…」→ 「仲間認識されない」
  - Cat: 「猫がのどを鳴らす理由…」→ 「細胞修復してた」
  - Hummingbird: 「この鳥、物理法則無視してる…」→ 「1秒80回」
- 期待形成→裏切りの構造でドーパミン予測誤差を最大化

### 8. 感情の伝染 (Emotional Contagion)

| 項目 | 内容 |
|---|---|
| **提唱者** | Hatfield et al. (1993) |
| **知見** | 他者の感情表現は自動的に観察者に転移する。音声の感情トーンも例外なく伝播 |
| **音声の感情伝播速度** | 200-500ms で観察者の感情状態に変化 |

**v4の音声感情分析:**
| 音声 | トーン | 感情 | 伝染結果 |
|---|---|---|---|
| メイン(Keita) | 淡々 | 中性 | △ 感情転移なし |
| ツッコミ(Nanami) | 少し速め | 呆れ | ○ 軽いわかり |

**問題点:**
- メイン音声が「淡々」としているため、感情の波が小さい
- ツッコミの「呆れ」だけが感情的高峰

**v5改善:**
- edge-ttsの **style パラメータ** を活用
  - メイン: `style="excited"` または `style="cheerful"` で感情注入
  - ツッコミ: `style="angry"` または `style="sad"` で感情強調
- 感情の起伏を作ることで感情伝染を最大化

| **時間分解能** | 2-5ms | 音の定位、ステレオ知覚 |
| **音素識別** | 20-50ms | 子音と母音の識別 |
| **リズム知覚** | 100-200ms | ビート、テンポの感知 |
| **事象認知** | 200-500ms | 「何かが起きた」の知覚 |
| **短期記憶** | 1-3秒 | 作業記憶の保持時間（音韻ループ） |
| **注意持続** | 3-8秒 | 現在のタスクへの注意維持 |
| **エピソード記憶** | 10秒以上 | 長期記憶への符号化 |

**v4の時間設計検証:**

| 1ネタ尺 | 5秒 | 3-8秒 (注意持続) | ○ 範囲内 |

### 10. ゲシュタルトの法則

| 法則 | 内容 | 動画への適用 |
|---|---|---|
| **近接の法則** | 近い要素はグループとして知覚 | テロップと説明文の距離を10px以内に |
| **類似の法則** | 似た要素はグループとして知覚 | 同じテーマの要素は同色で統一 |
| **閉合の法則** | 不完全な図形を補完して知覚 | テロップの「!?」で読者に想像させる |
| **連続性の法則** | 滑らかな連続を知覚 | プログレスバーは左→右の直線 |
| **図地の法則** | 前景と背景を分離 | テキストは背景と明確に区別 |

**v4のゲシュタルト分析:**
- ✅ 図地の法則: テキストに白縁取り（背景からの分離）
- ⚠️ 近接の法則: テロップと説明文の距離が離れすぎ（0.55と0.70 = 288px）
  → 15px以内に短縮すべき
- ⚠️ 類似の法則: ツッコミバルーンの色がシーンごとに違う → 統一感がない
  → 補色戦略はVon Restorff効果だが、ゲシュタルトの類似の法則と衝突
  → **トレードオフ**: 記憶には補色、統一感には同色。v5では補色を維持
